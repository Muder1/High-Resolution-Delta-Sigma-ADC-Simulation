# High-Resolution Delta-Sigma ADC: System Modeling & RTL Verification

## 📌 Project Overview

This repository contains the digital signal processing (DSP) models and synthesizable hardware architecture for a high-resolution **Delta-Sigma Analog-to-Digital Converter (ADC)**, developed for the **ISRO Inter IIT Tech Meet Problem Statement**.

The project implements a **3rd-Order Delta-Sigma Modulator** together with a **4th-Order Cascaded Integrator-Comb (CIC) Decimation Filter**. It demonstrates the complete workflow from floating-point MATLAB modeling to fixed-point RTL implementation, verifying that the Verilog hardware preserves the high-resolution performance of the theoretical DSP model.

---

## ⚙️ System Specifications

| Parameter | Value |
|-----------|-------|
| **Input Signal** | 80 Hz Sine Wave |
| **Modulator** | 3rd-Order CIFB (Cascade of Integrators with Distributed Feedback) |
| **Oversampling Frequency (Fs)** | 128 kHz |
| **Decimation Filter** | 4th-Order CIC Filter |
| **Decimation Factor (R)** | 256 (Default, 500 Hz Output) / 128 (Configurable) |
| **Internal Datapath** | 33-bit Accumulators |
| **RTL Output Width** | 20-bit Signed Integer |
| **MATLAB Performance** | 19.06 ENOB |
| **Verified RTL Performance** | 18.85 ENOB |

## 📁 Repository Structure

| File / Folder | Description |
|---------------|-------------|
| `main.m` | Main execution script. Defines the system parameters, performs the complete Delta-Sigma ADC simulation, exports the Verilog input bitstream, and computes the final hardware performance metrics. |
| `generateSignal.m` | Generates the input sine wave used throughout the simulation. |
| `Analysis/` | Collection of utility scripts that perform individual analysis tasks such as SNR calculation, ENOB calculation, FFT generation, plotting, and other performance evaluation functions. |
| `Modulators/` | Contains implementations of the 1st-order, 2nd-order, and 3rd-order Delta-Sigma modulators. |
| `Filters/` | Contains a generic CIC filter implementation capable of simulating CIC filters of different orders. |
| `CompareCICOrders.m` | Compares CIC filter orders (1 through 5) by computing the corresponding SNR and ENOB values for a fixed Nyquist sampling rate. |
| `CompareDecimationFactor.m` | Compares the effect of different decimation factors while keeping the CIC filter order fixed. |
| `VerilogTest.m` | Imports the hardware output generated by the Verilog simulation and computes the corresponding SNR and ENOB for verification against the MATLAB model. |
| `bitstream_in.txt` | One-bit PDM bitstream generated by the Delta-Sigma modulator and used as input to the Verilog simulation. |
| `rtl_output.txt` | Output samples generated by the Verilog simulation after CIC filtering. |

### SystemVerilog Architecture (Hardware)

| File | Description |
|------|-------------|
| `design.sv` | Synthesizable RTL implementation of the 4th-order CIC filter using a 33-bit internal datapath and 20-bit output truncation. |
| `testbench.sv` | Verification testbench that reads `bitstream_in.txt`, feeds the RTL, and writes hardware output to `rtl_output.txt`. |

## 🚀 Prerequisites

Before running the project, install the following tools:

- MATLAB
- Icarus Verilog (`iverilog`, `vvp`)

### Install Icarus Verilog (Fedora)

```bash
sudo dnf install iverilog
```

---

## 🛠️ Running the Co-Simulation

### Step 1 — Generate the Hardware Stimulus (MATLAB)

Run:

```matlab
main
```

This will:

- Generate the 80 Hz test signal
- Simulate the 3rd-order Delta-Sigma Modulator
- Produce a 1-bit PDM bitstream
- Export the bitstream to:

```
bitstream_in.txt
```

---

### Step 2 — Run the RTL Simulation

Ensure the following files are in the same directory:

```
design.sv
testbench.sv
bitstream_in.txt
```

Compile the design:

```bash
iverilog -g2012 -o sim design.sv testbench.sv
```

Execute the simulation:

```bash
vvp sim
```

The simulation processes the complete input bitstream and generates:

```
rtl_output.txt
```

which contains the raw 20-bit signed hardware output samples.

---

### Step 3 — Verify the Hardware (MATLAB)

Load `rtl_output.txt` into MATLAB and execute the verification section of the project to:

- Normalize the 20-bit hardware output
- Compute the FFT
- Calculate SNR
- Calculate ENOB
- Compare the RTL output against the floating-point MATLAB model

---

## 📊 Results

The co-simulation validates that the fixed-point hardware closely matches the theoretical DSP model.

| Metric | MATLAB Model | RTL Hardware |
|--------|-------------:|-------------:|
| ENOB | **19.06 bits** | **18.85 bits** |
| Internal Datapath | Floating Point | 33-bit Fixed Point |
| Output Resolution | Floating Point | 20-bit Signed |

### Key Observations

- Mathematical bit-growth analysis determined that **33-bit accumulators** safely prevent integrator overflow.
- Output truncation to **20 bits** introduces only a negligible quantization error.
- The RTL implementation preserves nearly all of the theoretical performance.
- The design comfortably exceeds the **16-bit ENOB** target specified in the ISRO Inter IIT problem statement.
